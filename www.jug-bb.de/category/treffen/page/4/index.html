Big Data in der Praxis

3. Oktober 2012 von Oliver Fischer in Treffen
Schlagwörter: big, bigdata, nosql

Die Java Usergroup Berlin-Brandenburg trifft sich am Montag den 22. Oktober 2012 ab 17 Uhr zum Thema Big Data in der Praxis. Drei Referenten bestreiten die Veranstaltung: Eberhard Wolff von adesso diskutiert neue Persistenz-Ansätze und NoSQL, Max Tritschler von Fraunhofer Fokus stellt eine Big-Data-Anwendung vor und Vishal Bagga von Versant zeigt, wie Versant Big Data and Java zusammenbringt.

Die anschließende Paneldiskussion thematisiert Fragen wie die Marktreife von Lösungen, mögliche Anwendungsszenarien und ähnliches – und gibt so Impulse, um das Thema Big Data in der Praxis im abschließenden Get-Together direkt untereinander und mit den Referenten zu vertiefen.

Die Vorträge

NoSQL – Neue Persistenz-Ansätze (Eberhard Wolff): Die Datenbank-Welt steht vor einer fundamentalen Änderung – ausgelöst durch die NoSQL-Datenbanken. Aber warum sind neue Technologien überhaupt notwendig? Dieser Vortrag gibt eine Einführung in NoSQL, einen Überblick über die verschiedenen Ansätze, die sich unter diesem Begriff verbergen und typische Anwendungsszenarien.

JPA the Versant way (Vishal Bagga): Abstract: Modern applications manage ever-larger data volumes and rapidly growing data complexity, a problem commonly referred to as Big Data. The data volume is a significant problem but it is the complexity of the data that is most difficult to handle. These challenges have lead to the rise of different technologies that adopt different approaches to data storage and retrieval. This presentation will introduce Versant JPA, a fast and scalable Java API and how it can be used to solve big data problems efficiently. The presentation will also introduce a Java based Big Data framework that allows you to to Ingest high volumes of data and also perform Analytics on this data.

Praxisbericht Fraunhofer FOKUS: Database Benchmark for IPTV (Max Tritschler): The number of users simultaneously accessing the same TV programs and channels via Internet is rapidly growing. In this application domain, the database must provide fast access to rather complex data, easy integration into an ever evolving application code, and support massive concurrent access from potentially millions of consumers.

Fraunhofer FOKUS benchmarked the use of Versant Object Database and Oracle’s MySQL within the Fraunhofer FOKUS Open IPTV Ecosystem. The benchmark provides quantitative results to help compare and analyze performance, complexity, scalability and total costs of ownership of the tested ecosystem.

GPars – Parallele Programmierung für Dich und mich

26. September 2012 von Oliver Fischer in Treffen
Schlagwörter: gpars, groovy, Treffen

Am 16. Oktober 2012 lädt die Java Usergroup Berlin-Brandenburg und die Hypoport AG zum Vortrag GPars – Parallele Programmierung für Dich und mich mit Dierk König ein.

Der Vortrag

GPars – Parallele Programmierung für Dich und mich (Dierk König): Seit jeher bietet die Java platform gute Unterstützung für die nebenläufige Programmierung. Die Bordmittel für multithreading und Synchronisation haben aber ihre Tücken. Dierk König stellt daher zusätzliche Konzepte für den vereinfachten Umgang mit Nebenläufigkeit vor: fork/join, map/reduce, Aktoren, Agenten und Datenflüsse. Der Beispielcode wird mit GPars vorgeführt, Groovys Standardbibliothek für Nebenläufigkeit.

Der Redner

Dierk König: Dierk König (JavaOne Rock Star) ist Fellow bei der Canoo Engineering AG in Basel. Er betreibt das Open-Source-Projekt Canoo WebTest und ist Committer in den Projekten Groovy, Grails, GPars und GroovyFX. Er publiziert und spricht auf internationalen Konferenzen zu Themen moderner Softwareentwicklung. Er ist Autor des Buchs “Groovy in Action”. Twitter: @mittie.

Database Change Management mit Liquibase

21. September 2012 von Oliver Fischer in Treffen

Am 10. Oktober 2012 lädt die Java Usergroup Berlin-Brandenburg und Ebay Tech-Talk Berlin zum Vortrag Database Change Management mit Liquibase mit Dirk Mahler von buschmais ein.

Der Vortrag

Database Change Management mit Liquibase (Dirk Mahler): “Am Tag nach der ersten Inbetriebnahme einer Anwendung taucht die Frage auf, wie ab sofort mit Änderungen am relationalen Datenbankschema umgegangen werden soll. Die naheliegende Antwort lautet: ein Migrationsskript muss her. Schnell entsteht daraus eine unüberschaubare Menge von Skripten und mindestens ebenso schnell geht die Übersicht darüber verloren, welche Skripte auf welche Datenbankinstanz bereits angewendet wurden bzw. noch angewendet werden müssen.

Mit Liquibase steht ein robustes und erprobtes Werkzeug zur Verfügung, welches ein hohes Maß an Automatisierung von Datenbankmigrationen auf Entwicklerarbeitsplätzen, Testsystemen bis hin zur Produktion ermöglicht. Der Vortrag erläutert die zugrundeliegenden Konzepte und veranschaulicht sie anhand praxisnaher Beispiele.”

Der Redner

Dirk Mahler: Dirk Mahler ist als Senior Consultant auf dem Gebiet der Java-Enterprise-Technologien tätig und setzt sich in seiner täglichen Arbeit mit einem breiten Spektrum an Themen rund um Softwarearchitektur auseinander. Er ist Gesellschafter der buschmais GbR, einem Beratungshaus mit Sitz in Dresden.

Unit Tests für SQL Queries – Daleq

31. Juli 2012 von Oliver Fischer in Treffen
Schlagwörter: daleq, Treffen, Vortrag

Am 9. August 2012 lädt die Java Usergroup Berlin-Brandenburg zusammen mit brands4friends zum Vortrag Unit Tests für SQL Queries – Daleq mit Lars Girndt von brands4friends ein.

Der Vortrag

Unit Tests für SQL Queries – Daleq (Lars Girndt): Es ist bis heute nicht trivial, wartbare Unit-Tests für SQL Queries zu schreiben. Eine Herausforderung dabei ist, aus Entwicklungssicht effizient Testdaten zu erzeugen. Wir haben bei brands4friends das Open Source Framework Daleq entwickelt, mit dem wir Testdaten in Java generieren. Daleq ist eine DSL, die als dünner Wrapper um DbUnit ermöglicht, pro Test individuell Daten zu beschreiben. Dieser Vortrag stellt unsere Konzepte für das Unit-Testen von SQL Queries im Allgemeinen dar und geht anschließend darauf ein, wie wir diese mit Daleq umsetzen. Daleq wurde von uns auf Github veröffentlicht: https://github.com/brands4friends/daleq

Der Redner

Lars Girndt: Lars Girndt arbeitet als Senior Developer bei brands4friends und ist dort als Experte von High Traffic Webseiten für die Architektur des Webshops verantwortlich. Neben dem selbstverständlichen Umgang mit modernen Java Technologien wie Spring oder der NoSQL Datenbank Redis gehört auch das kontinuierliche Skalieren der Plattform zu seinem Portfolio.

Aerogear: Mobile Lösungen mit Java EE und JBoss Technologien

5. Juli 2012 von Oliver Fischer in Treffen
Schlagwörter: Vortrag

Am 17. Juli 2012 lädt die Java Usergroup Berlin-Brandenburg zusammen mit akquinet AG zum Vortrag Aerogear: Mobile Lösungen mit Java EE und JBoss Technologien mit Heinz Wilming und Philipp Kumar ein.

Der Vortrag

Aerogear: Mobile Lösungen mit Java EE und JBoss Technologien (Heinz Wilming dund Philipp Jumar): Begriffe wie HTML 5, Mobile Web, Hybridapps, REST oder Cloud sind mittlerweile in aller Munde und die meisten Entwickler haben Erfahrung auf den einzelnen Themengebieten gesammelt. Nur, wie spielen diese zusammen? Wie lassen sich die zugehörigen Technologien zu einer mobile Gesamtlösung integrieren? Dieser Vortrag beantwortet diese Fragen und zeigt, wie sich eine Cloud-basierte mobile Applikation mit JBoss Technologien und Java EE Standards auf Basis einer Plain-Old-HTML5-Architektur (POH5) erstellen lässt.

Die Redner

Philipp Kumar: ist Leiter des Competence Center für mobile Lösungen bei der akquinet AG in Berlin. Er verfügt über langjährige Erfahrungen in der Durchführung von Projekten mit mobilen Klienten und ist spezialisiert auf Android-basierte Applikationen sowie deren Integration in Enterprise-Infrastrukturen.

Heinz Wilming: Heinz Wilming ist Leiter des JBoss Competence Center der akquinet AG. Er beschäftigt sich dort mit Technologien und Architekturen für verteilte Anwendungen. Sein Fokus liegt dabei auf der Java EE Plattform und Open Source Technologien.
