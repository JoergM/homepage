Mai 2012, findet der 3rd Eclipse Integrated Development Day – Eclipse IDD 2012 im Fraunhofer-Forum Berlin, Anna-Louisa-Karsch-Str. 2, 10178 Berlin, statt.

Model Driven Engineering (MDE) – auch modellgetriebene Entwicklung -verspricht, die zunehmende Komplexität innerhalb von Software und ihrem Entwicklungsprozess bei steigenden Abstraktionsniveau durch Modelle als Informationsspeicher zu beherrschen. Diesen Trend verstärken die Verfügbarkeit von stabilen wie umfangreichen Werkzeugen und unterstützende Plattformen wie die von Eclipse.

Der Eclipse Integrated Development Day in Berlin ist eine ganztägige Konferenz für technische Entwickler, Architekten und Manager aus Industrie, Forschung – aber auch für den Nachwuchs. Teilnehmern, die sich Werkzeug- und Prozessintegration ebenso interessieren wie für Automation in Entwicklungsprozessen, werden Vorträge und Live-Demonstrationen von Experten aus der Eclipse-Community und führender Unternehmen geboten.

  • Dieser Eclipse Day konzentriert sich unter anderem auf:
  • Application Lifecycle Management
  • Automatisierung und Integration in Entwicklungsprozessen
  • Zusammenarbeit von Entwicklern
  • Erfahrungsberichte

Aktuell werden interessante Beiträge für Präsentationen, Demos, Fallstudien oder auch Erfahrungsaustausch im Bereich der integrierten Entwicklung basierend auf der Eclipse-Technologie gesucht. Die Dauer der Präsentationen beträgt jeweils rund 30 Minuten inklusive Zeit für Fragen und Diskussionen.

Vorschläge können bis zum 30. März 2012 an EclipseDay@fokus.fraunhofer.de geschickt werden.

Neben dem Thema wird ein Abstrakt von 100 bis 200 Worten sowie ein Kurzportrait des Referenten inklusive Kontaktinformationen benötigt.

Weitere detaillierte Informationen unter: http://wiki.eclipse.org/Eclipse_IDD_2012

Am 08. Juli wird Shaun Smith uns einen Vortrag über EclipseLink halten.

Der Vortrag:

The founding of EclipseLink, the Eclipse Persistence Services Project, signals the inauguration of a new generation of Java persistence where all of the dominant persistence standards are implemented and accessible from a single source.

EclipseLink is composed of a set of separate runtime components that can be used in both clients and servers, and is the first open source project to offer a full suite of persistence technologies. By supporting all of the popular standards, including object-relational mapping using the Java Persistence API (JPA), object-to-XML mapping through the Java Architecture for XML Binding (JAXB), and Service Data Objects (SDO) that bridge the relational and XML worlds, EclipseLink provides the most comprehensive persistence platform available. Equally compelling is the fact that these technologies are offered with a rich array of advanced features and high-end performance options.

In this session, attendees will learn what EclipseLink is, the standard technologies it implements, and how these technologies can be used separately or together to meet advanced and diverse application requirements. We will show examples of how to use EclipseLink to map Java objects in object-relational and object-XML contexts, and how it can be used in any runtime environment, including OSGi, Eclipse RCP, Java SE, Java EE, Spring and Tomcat.

Der Sprecher:

Shaun Smith is the EclipseLink Ecosystem Development Lead, co-lead of the Eclipse Dali Java Persistence API (JPA) Tools Project, and a product manager for Oracle TopLink, Oracle’s JPA implementation and the basis of the open source ‘TopLink Essentials’ JPA Reference Implementation. He has been building systems using object-relational mapping technologies for almost a decade in both Java and Smalltalk in the banking, telecommunications, and energy industries. Shaun’s a frequent conference presenter and has recently spoken at JavaPolis, SpringOne, W-JAX, Eclipse Summit Europe, EclipseWorld, The Server Side Java Symposium, and JavaOne.

Für den Mac gibt es selbstverständlich alles zu dem Thema aber die Installation von CVS oder SubVersion ist mit Linux vergleichbar kryptisch und wer es noch nie gemacht hat wird seinen “Spass” damit haben.
Für SubVersion gibt es eine Distribution die genauso einfach wie jede andere Mac Anwendungen zu installieren geht – rein durch kopieren.

Der MAS = Mac SVN server.

Der MAS bringt einen eigenen Apache2 mit und eine Berkley-DB. Der Zugriff erfolgt über HTTP mittels WebDAV oder per Webinterface.

Die Nutzung ist denkbar einfach. Einfach die Anwendung MAS starten und los gehts. Man muss noch einen SVN-User anlegen (innerhalb der MAS-Anwendung) und mehr brauchts nicht.

Ich habe ihn mit Eclipse ausprobiert und er arbeitet seit einer Woche ohne jede Beanstandung.

Es sieht so aus als würde sich in diesem Projekt nichts mehr weiter tun. Deshalb wäre es sehr interessant mal die interne Struktur zu untersuchen und die Integration speziell von Subversion zu betrachten um neuere Versionen nachinstallieren zu können.
