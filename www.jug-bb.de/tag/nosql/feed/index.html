November 2012 lädt die Java Usergroup Berlin-Brandenburg und die Groupon GmbH zum Vortrag redis – mehr als ein Cache mit André Neubauer von der Deutsche Post E-Post Development GmbH ein.

Der Vortrag

redis – mehr als ein Cache (André Neubauer): redis wird häufig vor allem als Alternative zu memcached verstanden. Dabei lässt sich der Key-Value-Store nicht nur als Cache verwenden, sondern ist weitaus mächtiger. Der Talk geht auf die verschiedenen Besonderheiten (u. a. Datatypes, Transactions, Replication) des NoSQL-Stores ein und stellt die Applikationsintegration mittels Spring Data Redis vor.

Der Redner

André Neubauer: André Neubauer ist Head of Development bei der E-Post in Berlin. Sein Interesse gilt dem Thema Lean Product Development und damit einhergehende Aspekte der Softwareentwicklung und -architektur. In seiner Freizeit ist er nach wie vor Software-Enthusiast und beschäftigt sich gerne mit State-of-the-Art-Technologien.

]]> http://www.jug-bb.de/2012/11/redis-mehr-als-ein-cache/feed/ 0 Big Data in der Praxis http://www.jug-bb.de/2012/10/big-data-in-der-praxis/ http://www.jug-bb.de/2012/10/big-data-in-der-praxis/#comments Tue, 02 Oct 2012 23:11:04 +0000 http://www.jug-bb.de/?p=930 Die Java Usergroup Berlin-Brandenburg trifft sich am Montag den 22. Oktober 2012 ab 17 Uhr zum Thema Big Data in der Praxis. Drei Referenten bestreiten die Veranstaltung: Eberhard Wolff von adesso diskutiert neue Persistenz-Ansätze und NoSQL, Max Tritschler von Fraunhofer Fokus stellt eine Big-Data-Anwendung vor und Vishal Bagga von Versant zeigt, wie Versant Big Data and Java zusammenbringt.

Die anschließende Paneldiskussion thematisiert Fragen wie die Marktreife von Lösungen, mögliche Anwendungsszenarien und ähnliches – und gibt so Impulse, um das Thema Big Data in der Praxis im abschließenden Get-Together direkt untereinander und mit den Referenten zu vertiefen.

Die Vorträge

NoSQL – Neue Persistenz-Ansätze (Eberhard Wolff): Die Datenbank-Welt steht vor einer fundamentalen Änderung – ausgelöst durch die NoSQL-Datenbanken. Aber warum sind neue Technologien überhaupt notwendig? Dieser Vortrag gibt eine Einführung in NoSQL, einen Überblick über die verschiedenen Ansätze, die sich unter diesem Begriff verbergen und typische Anwendungsszenarien.

JPA the Versant way (Vishal Bagga): Abstract: Modern applications manage ever-larger data volumes and rapidly growing data complexity, a problem commonly referred to as Big Data. The data volume is a significant problem but it is the complexity of the data that is most difficult to handle. These challenges have lead to the rise of different technologies that adopt different approaches to data storage and retrieval. This presentation will introduce Versant JPA, a fast and scalable Java API and how it can be used to solve big data problems efficiently. The presentation will also introduce a Java based Big Data framework that allows you to to Ingest high volumes of data and also perform Analytics on this data.

Praxisbericht Fraunhofer FOKUS: Database Benchmark for IPTV (Max Tritschler): The number of users simultaneously accessing the same TV programs and channels via Internet is rapidly growing. In this application domain, the database must provide fast access to rather complex data, easy integration into an ever evolving application code, and support massive concurrent access from potentially millions of consumers.

Fraunhofer FOKUS benchmarked the use of Versant Object Database and Oracle’s MySQL within the Fraunhofer FOKUS Open IPTV Ecosystem. The benchmark provides quantitative results to help compare and analyze performance, complexity, scalability and total costs of ownership of the tested ecosystem.
