Java One und JEE 7

20. November 2012 von Oliver Fischer in Treffen
Schlagwörter: Treffen

Am 29. November 2012 lädt die Java Usergroup Berlin-Brandenburg und die Hypoport AG zum Vortrag Java One und JEE 7 Update mit Wolfgang Weigend und Peter Dowschkinow von der Oracle Deutschland ein.

Die Vorträge

JavaOne Update zur Java Plattform (Wolfgang Weigend): Nach der JavaOne sind die Weichen für das JDK 8 in 2013 gestellt, und Java SE 8 beinhaltet Lambda-Ausdrücke, eine verbesserte Interoperabilität von JavaScript zur JVM mit der JavaScript-Engine Nashorn und das überarbeitete Date & Time API im JSR-310. Die HotSpot-basierte JVM wird kein PermGen mehr enthalten und die konvergierenden JVM-Merkmale sollen im Zeitraum vom JDK 8 eingebaut sein.
Mit dem JSR-308 Annotationen für Java Typen wird die Syntax der Java Programmiersprache erweitert, um die Verwendung von Type-Name und Deklaration kennzeichnen zu können. JavaFX wird als strategische Java Desktop Technologie in Java SE 8 einfliessen und dann gemeinsam mit dem JDK 8 ausgeliefert. Der standardisierte Übergang im Rahmen von JavaFX zu einer eigenen Spezifikation erfolgt über das Projekt OpenJFX. Die Konvergenz von Java SE und CDC Embedded wird weiter vorangetrieben. Mit der Einführung von Java SE 8 Compact Profilen werden drei neue Profile als Subsets von Java SE im JDK 8 zur Verfügung stehen, um die Modularisierung für Java Embedded im JDK 8 für Jisaw vorzubereiten. Die Java Plattformmodularisierung mit dem Projekt Jigsaw ist für das JDK 9 vorgesehen.

JEE 7 (Peter Doschkinow): Die Java EE 7 Plattform Spezifikation ist seit ihrer einstimmigen Annahme im März 2011 durch das Executive Committee für Java SE/EE in Arbeit. Ihre Fertigstellung ist für die Mitte des nächsten Jahres geplant. Nachdem im August dieses Jahres die Entscheidung getroffen wurde,
das zentrale PaaS-Thema aus dem Umfang von Java EE 7 herauszunehmen und in die Java EE 8 zu verschieben,
konzentrieren sich die Anstrengungen im Java EE Umfeld auf die weitere Verbesserung der Produktivität und eine umfassende Unterstützung für HTML 5. Der Vortrag gibt eine Übersicht über den aktuellen Status der einzelnen Java EE 7 Teilspezifikationen und geht auf einige ihrer neuen Features ein. Es wird ein Ausblick auf die Java EE 8 gemacht und das mit Java EE in Verbindung stehende Projekt Avatar vorgestellt.

Die Redner

Wolfgang Weigend: Wolfgang Weigend, Systemberater für die Oracle Fusion Middleware bei der Oracle Deutschland B.V. & Co. KG, zuständig für Architektur, Java Technologie und strategischen Produkteinsatz bei Großkunden, verfügt über langjährige Erfahrung in der Systemberatung und im Bereich objektorientierter Softwareentwicklung. Davor war er als Principal Systems Engineer 9,5 Jahre bei der BEA Systems GmbH für strategische Kunden tätig und koordinierte gleichzeitig als Teamleader Systems Engineering alle Systemberater in der Central Region Deutschland, Österreich und Schweiz.
Wolfgang Weigend studierte an der FH Darmstadt Elektrotechnik/Automatisierungstechnik mit dem Studienschwerpunkt Datentechnik. Bevor der Diplom-Ingenieur 1999 zu BEA Systems kam, war er als Systemberater für Oracle, Texas Instruments Software und Sun Microsystems tätig.

Peter Dowschkinow: Peter Doschkinow arbeitet als Senior Java Architekt bei Oracle Deutschland. Er beschäftigt sich mit serverseitigen Java-Technologien und Frameworks, Web Services und Business Integration, die er in verschiedenen Kundenprojekten erfolgreich eingesetzt hat. Vor seiner Tätigkeit bei Oracle hat er wertvolle Erfahrungen als Java Architect and Consultant bei Sun Microsystems gesammelt. Peter Doschkinow besitzt mehr als 20 Jahre Berufserfahrung in der IT-Industrie und ist Sun Certified Architect, Developer und Programmer.

redis – mehr als ein Cache

15. November 2012 von Oliver Fischer in Treffen
Schlagwörter: nosql, redis, Treffen, Vortrag

Am 21. November 2012 lädt die Java Usergroup Berlin-Brandenburg und die Groupon GmbH zum Vortrag redis – mehr als ein Cache mit André Neubauer von der Deutsche Post E-Post Development GmbH ein.

Der Vortrag

redis – mehr als ein Cache (André Neubauer): redis wird häufig vor allem als Alternative zu memcached verstanden. Dabei lässt sich der Key-Value-Store nicht nur als Cache verwenden, sondern ist weitaus mächtiger. Der Talk geht auf die verschiedenen Besonderheiten (u. a. Datatypes, Transactions, Replication) des NoSQL-Stores ein und stellt die Applikationsintegration mittels Spring Data Redis vor.

Sein Interesse gilt dem Thema Lean Product Development und damit einhergehende Aspekte der Softwareentwicklung und -architektur. In seiner Freizeit ist er nach wie vor Software-Enthusiast und beschäftigt sich gerne mit State-of-the-Art-Technologien.

GPars – Parallele Programmierung für Dich und mich

26. September 2012 von Oliver Fischer in Treffen
Schlagwörter: gpars, groovy, Treffen

Am 16. Oktober 2012 lädt die Java Usergroup Berlin-Brandenburg und die Hypoport AG zum Vortrag GPars – Parallele Programmierung für Dich und mich mit Dierk König ein.

Der Vortrag

GPars – Parallele Programmierung für Dich und mich (Dierk König): Seit jeher bietet die Java platform gute Unterstützung für die nebenläufige Programmierung. Die Bordmittel für multithreading und Synchronisation haben aber ihre Tücken. Dierk König stellt daher zusätzliche Konzepte für den vereinfachten Umgang mit Nebenläufigkeit vor: fork/join, map/reduce, Aktoren, Agenten und Datenflüsse. Der Beispielcode wird mit GPars vorgeführt, Groovys Standardbibliothek für Nebenläufigkeit.

Dierk König: Dierk König (JavaOne Rock Star) ist Fellow bei der Canoo Engineering AG in Basel. Er betreibt das Open-Source-Projekt Canoo WebTest und ist Committer in den Projekten Groovy, Grails, GPars und GroovyFX. Er publiziert und spricht auf internationalen Konferenzen zu Themen moderner Softwareentwicklung. Er ist Autor des Buchs “Groovy in Action”. Twitter: @mittie.

Unit Tests für SQL Queries – Daleq

31. Juli 2012 von Oliver Fischer in Treffen
Schlagwörter: daleq, Treffen, Vortrag

Am 9. August 2012 lädt die Java Usergroup Berlin-Brandenburg zusammen mit brands4friends zum Vortrag Unit Tests für SQL Queries – Daleq mit Lars Girndt von brands4friends ein.

Der Vortrag

Unit Tests für SQL Queries – Daleq (Lars Girndt): Es ist bis heute nicht trivial, wartbare Unit-Tests für SQL Queries zu schreiben. Eine Herausforderung dabei ist, aus Entwicklungssicht effizient Testdaten zu erzeugen. Wir haben bei brands4friends das Open Source Framework Daleq entwickelt, mit dem wir Testdaten in Java generieren. Daleq ist eine DSL, die als dünner Wrapper um DbUnit ermöglicht, pro Test individuell Daten zu beschreiben. Dieser Vortrag stellt unsere Konzepte für das Unit-Testen von SQL Queries im Allgemeinen dar und geht anschließend darauf ein, wie wir diese mit Daleq umsetzen. Daleq wurde von uns auf Github veröffentlicht: https://github.com/brands4friends/daleq

Der Redner

Lars Girndt: Lars Girndt arbeitet als Senior Developer bei brands4friends und ist dort als Experte von High Traffic Webseiten für die Architektur des Webshops verantwortlich. Neben dem selbstverständlichen Umgang mit modernen Java Technologien wie Spring oder der NoSQL Datenbank Redis gehört auch das kontinuierliche Skalieren der Plattform zu seinem Portfolio.
