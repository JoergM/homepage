eCOMM-Seminare Brandenburg

16. Juni 2007 von Ralph Bergmann in Allgemein
Schlagwörter: Sun, Treffen


eCOMM Seminar: Kosten sparen mit elektronischen Rechnungen
Mittwoch, 20. Juni 2007; 17.00-20.15 Uhr; IHK Potsdam

Seit 2004 gilt das neue Umsatzsteuerrecht. An viele Regeln hat man
sich gewöhnt. Oftmals werden jedoch die Anforderungen an “die Echtheit
der Herkunft und die Unversehrtheit des Inhalts” der Rechnung (Zitat
UStG) bei “auf elektronischem Weg übermittelten Rechnungen” übersehen.
Online-Händler, Telekommunikationsunternehmen, Internet-Provider
stehen als Beispiele für Unternehmen, die gerne elektronische
Rechungen erstellen. Eingangsrechnungen müssen entsprechend bewertet
werden, damit kein unberechtigter Vorsteuerabzug erfolgt.
Auf der anderen Seite können sich Kosten- und Laufzeitvorteile für
Unternehmen in anderen Bereichen ergeben. Ein Handwerker, der seine
Rechnung elektronisch bei der Wohnunggesellschaft vorlegen kann, hat
unter Umständen zusätzliche Vorteile.
Dieses Seminar orientiert sich an der Praxis, Ihr Dozent ist
gelernter Kaufmann und Unternehmer und musste für sein Unternehmen
entsprechende Entscheidungen treffen.

http://www.ecomm-online.de/veranstaltung+M5f40f6777a0.html


eCOMM Seminar: Digital ist besser I – Einführung in die digitale
Fotografie | Mittwoch, 27. Juni 2007; 17.00-20.15 Uhr; IHK Potsdam

Das Seminar wird eine Einführung in die Welt der digitalen Fotografie
geben. Dabei werden alle relevanten Aspekte der Thematik praxisnah
und anschaulich beleuchtet. Den Teilnehmern soll ein Basiswissen
vermittelt werden, das es ihnen ermöglicht, selbstständig und
erfolgreich ansprechende Fotos aufzunehmen, die druck- und
internetfähig aufbereitet werden können.

http://www.ecomm-online.de/veranstaltung+M5635d6f130c.html


eCOMM Seminar: Digital ist besser II – Einführung in die digitale
Bildbearbeitung| Montag, 2. Juli 2007; 17.00-20.15 Uhr; IHK Potsdam

Aufbauend auf dem Seminar “Digital ist besser I” werden den
Teilnehmern Kenntnisse in der digitalen Bildbearbeitung vermittelt,
die es ihnen ermöglichen sollen, digitale Fotos professionell zu
bearbeiten und werbewirksam für Internet und Printmedien aufzu-
bereiten.

http://www.ecomm-online.de/veranstaltung+M5f691db3fcb.html


eCOMM Seminar: Digital ist besser III – Digitale Bildbearbeitung mit
Photoshop | Mittwoch, 11. Juli 2007; 17.00-20.15 Uhr; IHK Potsdam

Aufbauend auf dem Seminar “Digital ist besser II – Einführung in die
digitale Bildbearbeitung” soll dieses Seminar einen vertiefenden
Einblick in die Möglichkeiten der digitalen Bildbearbeitung mit
Photoshop bieten. Ziel ist es hierbei, den Teilnehmern grundlegende
Fertigkeiten zu vermitteln, die es ihnen ermöglichen, selbstständig
alltägliche Probleme der Bildbearbeitung zu lösen.

http://www.ecomm-online.de/veranstaltung+M567ce5f5a14.html

Fit im Cyberspace

6. Juni 2007 von Ralph Bergmann in Off-Topic
Schlagwörter: Berlin

Fraunhofer FIRST lädt zum virtuellen Ausdauertraining ein

“In der Mittagspause ein Verdauungsspaziergang durch Berlin Mitte, ohne das Bürogebäude zu verlassen – virtuelle Realität macht das schon lange möglich. Herkömmliche Spaziergänge im Cyberspace entsprechen also nicht den menschlichen Gewohnheiten und sind ungeeignet, den Bürospeck zu bekämpfen.

java.text.MessageFormat

6. Juni 2007 von Ralph Bergmann in Java
Schlagwörter: Java

In dem Standard-Package von „java.text“ findet man eine Reihe von kleinen Perlen für die tägliche Arbeit mit Texten. Eine ist z.B. „java.text.MessageFormat“ mit der man sehr leicht individualisierte Massen-Emails erzeugen kann. Hier ein kurzes Beispiel:

Source-Code:

package nice;
import java.text.MessageFormat;
import java.util.Date;

public final class DemoMessageFormat

private static final String MSG4FRIEND = “Hallo {1},\nhier ein kurzes Beispiel für die MessageFormat-Klasse.\nDieser Tipp ist vom {4, date}.\n\n”;
private static final String MSG4OTHER= “Guten Tag {0} {2},\nhier ein kurzes Beispiel für die MessageFormat-Klasse.\nDiese Nachricht ist vom {4, date}.\n\n”;
private static final Integer FREUND = new Integer(0);
private static final Integer KUNDE = new Integer(1);

private static final Object[][] USER = {
{ “Herr”, “Thomas”, “Tanne”, FREUND, new Date()},
{ “Frau”, “Elke”, “Eiche”, KUNDE, new Date() },
{ “Herr”, “Bernd”, “Buche”, KUNDE, new Date() },
{ “Herr”, “Ben”, “Birke”, FREUND, new Date() }
};

public static void main(String[] args) {

final MessageFormat mf4Freunde = new MessageFormat(MSG4FRIEND);
final MessageFormat mf4Kunden = new MessageFormat(MSG4OTHER);

for(int i = 0; i USER.length; i++) {
// MessageFormat für Freunde oder Kunden bestimmen.
final MessageFormat mf;
if (USER[i][3].equals(FREUND)) {
mf = mf4Freunde;
} else {
mf = mf4Kunden;
}
System.out.println(mf.format(USER[i]));
}
}
}
